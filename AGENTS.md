# Repo rules for AI coding agents

Read [`SESSION_HANDOFF.md`](SESSION_HANDOFF.md) first for project context. The rules below apply to every Devin / agent session on this repo and override the agent's own defaults.

## No CI

Do **not** set up or wait on CI for this repo.

- Do not add GitHub Actions workflows, pre-commit hooks, or any other CI / lint / test automation unless this file is updated to allow it.
- After creating a PR, do not poll CI status (e.g. `git(action="pr_checks")`). There are no checks configured and there should not be — opening a PR is the end of the workflow, not the start of a CI wait.
- Do not recommend setting up CI as an "optional next step" in PR descriptions, comments, or messages to the user. Leave it out entirely.
- The environment blueprint must not include CI-running commands.

If the user explicitly asks for CI in a specific session, that overrides this rule for that session only — do not edit this file to flip the default unless the user asks for the rule itself to change.

## Other defaults

- Communicate with the user in **Russian**. Code, commits, and PR bodies stay in English unless asked otherwise.
- Never commit secrets. `FIREWORKS_API_KEY` and `FLY_API_TOKEN` are provided via env / Devin secrets, never inline.
- Don't push directly to `main`; open a PR.
